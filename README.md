# SQL

## 1. 项目概述

本仓库仅含脚本 `MSQL命令代码记录.PY`（扩展名为 `.PY`，内容为教材习题对应的 **T-SQL 建表与约束** 语句与注释，以 `#` 与三引号字符串保存）。用于对照《数据库系统》类课本页码记录 `CREATE TABLE`、`ALTER TABLE`、`CHECK` 约束、`FOREIGN KEY` 等写法；**并非可执行的 Python 业务程序**，也未封装数据库连接代码。

## 2. 技术栈

### 前端

无。

### 后端

无应用层代码。

### 数据库

- Microsoft SQL Server 体系语法（`nchar`、`nvarchar`、`smalldatetime`、`getdate()` 等，出现在注释与字符串中的脚本）

### 开发工具与运维

- Python 文件作为**纯文本容器**保存 SQL（可用任意编辑器阅读）
- 实际执行环境：SQL Server Management Studio 或 `sqlcmd` 等客户端

## 3. 项目架构

单文件、**线性笔记结构**：

- 上行注释标明教材页码与题号。
- 对应 SQL 以 `#` 注释行或三引号块形式保存，便于复制到数据库客户端执行。

无模块划分、无运行时数据流。

## 5. 核心文件说明

### 项目入口文件和配置文件

- `MSQL命令代码记录.PY`：唯一核心文件；无 `if __name__` 入口逻辑，内容为注释与 SQL 字符串。

### 核心业务逻辑实现

- **图书/书店/销售表**：约第 58 页起，含主键、外键与销售数量约束示例。
- **Student/Course/SC/Teacher**：约第 60 页起，多表与 `CHECK`、`UNIQUE`、`DEFAULT` 组合。
- **后续小题**：如为 `Student` 增删列、`SC` 成绩范围约束等，均以 `ALTER TABLE` / `ADD CONSTRAINT` 形式出现。

### 数据模型和 API 接口

- **数据模型**：关系型表结构定义即模型；无 ORM。
- **API**：无；对外为 SQL DDL/DML 文本。

### 关键组件和服务模块

- 仅有“按题号分段的 SQL 片段”；执行时请在测试库中运行，避免影响生产。
